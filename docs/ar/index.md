# Fitcloak

خادم معاينة محلي خفيف الوزن لتطوير **قوالب Keycloak الأصلية بتقنية FreeMarker** — بدون Docker أو قاعدة بيانات أو تشغيل نسخة Keycloak كاملة.

## كيف يعمل

Fitcloak هو خادم محلي **يعمل بشكل منفصل عن Keycloak**. المطلوب فقط Java 17+.

1. تقوم بإنشاء أو تعديل قالب — قوالب `.ftl` القياسية، CSS، JS
2. يقوم Fitcloak بتجميع القوالب باستخدام نفس سلسلة الوراثة في Keycloak (`Base → Parent → Child`)
3. يستبدل بيانات الاختبار ويعرض الصفحة المُصيَّرة في متصفحك
4. عندما يصبح القالب جاهزاً — انسخه إلى Keycloak واختره في لوحة الإدارة
5. اختيارياً، يمكنك استخدام أدوات التجميع (Vite، Webpack، إلخ) لـ HMR (الاستبدال الساخن للوحدات) للتطوير الأسرع مع أي إطار عمل ويب

## لماذا Fitcloak؟

تخصيص صفحات تسجيل الدخول/الحساب/البريد الإلكتروني في Keycloak يعني عادةً دورة تطوير مؤلمة: إعادة بناء ملف JAR، إعادة تشغيل Keycloak، مسح الذاكرة المؤقتة، تحديث الصفحة. Fitcloak يزيل كل ذلك — فقط احفظ ملفك وشاهد النتيجة.

**خذ أي قالب Keycloak، وجّه Fitcloak إليه، وابدأ البرمجة.** البروكسي المدمج لخادم التطوير يتيح لك استخدام أي أداة واجهة أمامية — Vite أو Webpack أو Parcel — مع أي إطار عمل أو معالج مسبق: React أو Vue أو Svelte أو SCSS أو Tailwind، أياً كان ما تفضله. يعرض FreeMarker هيكل الصفحة، وأدواتك تتولى الواجهة الأمامية، وHMR يبقي دورة التطوير فورية.

هذا يمنحك المرونة الكاملة لتطوير الواجهات الأمامية الحديثة مع البقاء ضمن نظام القوالب الأصلي لـ Keycloak: لا إضافات SPI مخصصة، لا ارتباط بمورّد معين — فقط قوالب `.ftl` القياسية التي تُنشر على أي نسخة Keycloak كما هي.

## الميزات

- **استجابة فورية** — حرّر `.ftl` / `.css` / `.properties`، حدّث المتصفح
- **وراثة كاملة** — يحاكي سلسلة قوالب Keycloak `Base -> Parent -> Child`
- **تكامل Vite/HMR** — بروكسي لخادم التطوير للاستبدال الساخن للوحدات
- **اختبار ديناميكي** — تجاوز أي متغير قالب عبر معاملات URL
- **لوحة تحكم** — متصفح قوالب مع تصور الوراثة وروابط ضمان الجودة
- **بدون بنية تحتية** — فقط Java و Gradle، لا شيء آخر


### Fitcloak مقابل Keycloakify

| | Fitcloak | [Keycloakify](https://github.com/keycloakify/keycloakify) |
|---|---|---|
| **المنهج** | قوالب FreeMarker الأصلية + أي أداة واجهة أمامية | مكونات React تُترجم إلى قوالب |
| **حالة الاستخدام** | تخصيص قوالب Keycloak القياسية بتجربة تطوير حديثة | بناء واجهات مستخدم جديدة بالكامل باستخدام React |
| **الواجهة الأمامية** | أي إطار عمل (React، Vue، Svelte، Alpine.js، vanilla) أو بدون إطار | React/TypeScript فقط |
| **منحنى التعلم** | تعرف FreeMarker = جاهز للعمل | يتطلب معرفة بـ React/TypeScript |
| **المخرجات** | مجلد قالب قياسي (يعمل على أي Keycloak) | ملف JAR مع تطبيق React مُترجم |

Keycloakify يسلك مساراً مختلفاً: يستبدل FreeMarker بالكامل بتطبيق React أحادي الصفحة (SPA) وله خط بناء خاص به. Fitcloak يعمل مع نظام القوالب القياسي لـ Keycloak — نفس قوالب `.ftl`، نفس طريقة النشر، فقط سير عمل تطوير أفضل بكثير.
