# Fitcloak

Легковесный локальный сервер предварительного просмотра для разработки **нативных тем Keycloak на FreeMarker** — без Docker, базы данных или запуска полного экземпляра Keycloak.

[Начать работу](./quick-start)
[Посмотреть на GitHub](https://github.com/msotnikov/fitcloak)

---

## Зачем Fitcloak?

Кастомизация страниц логина/аккаунта/email в Keycloak обычно означает мучительный цикл обратной связи: пересобрать JAR, перезапустить Keycloak, очистить кеши, обновить страницу. Fitcloak устраняет всё это — просто сохраните файл и увидите результат.

**Возьмите любой шаблон Keycloak, укажите на него Fitcloak и начинайте работу.** Встроенный прокси dev-сервера позволяет использовать любой фронтенд-инструментарий — Vite, Webpack, Parcel — с любым фреймворком или препроцессором: React, Vue, Svelte, SCSS, Tailwind — что угодно. FreeMarker рендерит структуру страницы, ваши инструменты обрабатывают фронтенд, а HMR обеспечивает мгновенную обратную связь.

Это даёт вам полную гибкость современной фронтенд-разработки, оставаясь в рамках нативной системы тем Keycloak: никаких кастомных SPI, никакой привязки к вендору — только стандартные `.ftl`-шаблоны, которые деплоятся на любой экземпляр Keycloak как есть.

### Fitcloak vs Keycloakify

| | Fitcloak | [Keycloakify](https://github.com/keycloakify/keycloakify) |
|---|---|---|
| **Подход** | Нативные шаблоны FreeMarker + любой фронтенд-инструментарий | React-компоненты, компилируемые в темы |
| **Сценарий использования** | Кастомизация стандартных тем Keycloak с современным DX | Создание полностью новых UI на React |
| **Фронтенд** | Любой фреймворк (React, Vue, Svelte, Alpine.js, vanilla) или без него | Только React/TypeScript |
| **Порог входа** | Знаете FreeMarker — готовы к работе | Требуется знание React/TypeScript |
| **Результат** | Стандартная директория темы (работает на любом Keycloak) | JAR с скомпилированным React-приложением |

Keycloakify идёт другим путём: полностью заменяет FreeMarker на React SPA и имеет собственный пайплайн сборки. Fitcloak работает со стандартной системой тем Keycloak — те же `.ftl`-шаблоны, тот же деплой, просто гораздо лучший рабочий процесс разработки.

## Возможности

- **Мгновенная обратная связь** — редактируйте `.ftl` / `.css` / `.properties`, обновите браузер
- **Полное наследование** — эмулирует цепочку тем Keycloak `Base -> Parent -> Child`
- **Интеграция с Vite/HMR** — прокси к dev-серверу для горячей замены модулей
- **Динамическое тестирование** — переопределяйте любую переменную шаблона через параметры URL
- **Панель управления** — браузер шаблонов с визуализацией наследования и QA-ссылками
- **Без инфраструктуры** — только Java и Gradle, больше ничего
